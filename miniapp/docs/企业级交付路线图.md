# 智慧食堂小程序企业级交付路线图

本文档总结当前微信小程序学生端的状态、与企业级交付之间的差距，以及下一步最该做的工作。

## 当前完成情况

当前小程序已完成：

- `uni-app` 学生端工程：`miniapp/src`
- 新导航架构：`首页 / 菜单 / 健康 / 我的`
- 登录闭环：微信一键登录、账号密码登录、演示账号
- 首页、菜品检索、健康推荐、拍照识餐、智能顾问
- 与后端 `/api/*` 共用学生端能力
- iconfont.cn 风格图标替换 emoji
- 轻食绿色主题、毛玻璃和微动效打磨
- `npm run build:miniapp` 持续通过
- Web 静态视觉预览：`miniapp/preview.html`

## 与企业级交付的主要差距

### 1. 账号与登录还不够正式
当前仍以演示逻辑为主。

需要补齐：

- 真实微信小程序 AppID / AppSecret 接入
- 隐私协议、用户协议确认页
- 登录异常提示和重试策略
- 失效 token 自动刷新或踢出
- `profile` 页展示真实学生身份字段

### 2. 前端工程还没到交付标准
当前能运行，但不是企业级前端标准。

需要补齐：

- ESLint / Prettier 配置统一
- `uni -p h5` 预览或替代预览方案完全稳定
- 页面骨架屏 / loading / 空态统一
- 错误边界和接口异常 UI
- 图片懒加载、列表虚拟化、大页面性能检查
- 本地缓存、请求取消、防抖、幂等保存

### 3. 产品闭环还不完整
现在更像“功能集合”，不像完整产品。

需要补齐：

- 从首页到点餐/选菜的连续路径
- 菜品详情更完整：库存、今日供应、价格单位、供应时段
- 评价成功后的状态变化和回流
- 健康目标保存后更明确的页面反馈
- 拍照识餐失败兜底更自然
- `我的`页面补充设置、偏好、反馈入口

### 4. 设计与视觉还没达到稳定交付面
当前是“接近成熟小程序”，但不是最终交付版。

需要补齐：

- 真实食物图片或更高质量视觉资源
- 统一字号、间距、卡片高度规范
- 关键页面状态图：空态、加载态、错误态、成功态
- 深色模式可选，至少先做适配检查
- 页面转场和重点按钮交互更一致

### 5. 测试和质量门禁不足
这是企业级交付最关键的短板。

需要补齐：

- 小程序核心流程自动化测试
- 登录、推荐、菜品检索、顾问、识餐回归用例
- 后端契约测试和小程序请求联调测试
- 构建后产物校验
- CI 中加入小程序构建检查
- 真机验证 checklist

### 6. 配置与上线流程未固化
当前还依赖手动配置。

需要补齐：

- 小程序环境变量模板和发布配置文档
- 真实 AppID 注入
- 后端生产域名、HTTPS、微信合法域名配置
- 小程序体验版 / 正式版发布流程
- 灰度发布与回退策略
- 版本号、更新日志、发布产物归档

### 7. 安全与合规还差关键几步
当前已有基础安全，但微信小程序上线前还差：

- 隐私协议与授权弹窗
- 用户数据删除/导出能力评估
- 接口敏感信息校验加固
- 小程序审核常见拒绝点排查
- 真实存储和生产密钥管理

## 建议的下一步顺序

按“上线价值最高、阻断性最强”排序：

### 第一步：补齐账号与真实小程序接入
目标：

```text
从演示小程序变成可提交微信审核的真实小程序
```

内容：

```text
1. 配置真实微信 AppID / Secret
2. 完成微信登录生产闭环
3. 补隐私协议、授权确认
4. 稳定 token 刷新和未登录拦截
```

### 第二步：补齐产品主路径闭环
目标：

```text
用户从打开到完成核心动作不断裂
```

内容：

```text
1. 首页 → 菜单 → 菜品详情 → 选择/评价
2. 首页 → 健康 → 推荐 / 识餐 / 顾问
3. 我的 → 偏好/档案 → 影响推荐结果
4. 关键异常页和兜底状态补齐
```

### 第三步：补齐前端工程治理
目标：

```text
代码像交付项目，不像探索项目
```

内容：

```text
1. lint/format 规则
2. 页面状态标准化组件
3. 公共请求层统一
4. 性能与缓存治理
```

### 第四步：补齐测试与回归
目标：

```text
改功能不怕回归，发布前有门禁
```

内容：

```text
1. 核心接口联调测试
2. 学生端关键路径自动测试
3. 构建产物检查
4. 真机测试 checklist
```

### 第五步：补齐发布与运维文档
目标：

```text
别人也能按文档发布和维护
```

内容：

```text
1. 小程序发布流程
2. 环境配置说明
3. 后端 API 域名与微信后台配置说明
4. 常见审核问题与解决方案
```

## 当前最推荐的优先级

如果目标是“尽快接近企业级可交付”，我建议下一步只盯这 5 件事：

```text
1. 真实微信登录闭环
2. 首页/菜单/健康/我的主路径闭环
3. 关键页面状态治理
4. 小程序构建与核心测试门禁
5. 发布流程文档化
```

## 结论

当前小程序已经过了“能看、能跑、能演示”的阶段。  
下一步不是继续堆页面，而是把它从“功能原型”转成“可上线产品”。优先级应是：

```text
真实账号接入 > 产品闭环 > 工程治理 > 测试回归 > 发布流程
```
