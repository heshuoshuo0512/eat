# 智慧食堂微信小程序上线交付清单

## 1. 账号与密钥

- 微信公众平台已创建小程序并完成主体认证。
- `miniapp/src/manifest.json` 的 `mp-weixin.appid` 已替换为真实 AppID。
- `miniapp/project.config.json` 的 `appid` 已替换为真实 AppID。
- 服务端只在运行环境配置 `WECHAT_MINIAPP_APPID`、`WECHAT_MINIAPP_SECRET`、`WECHAT_LOGIN_TIMEOUT_MS`。
- 小程序源码、构建产物和前端环境变量中不出现 AppSecret、AI Key、数据库密码。

## 2. 域名与网络

- 后端 API 使用 HTTPS 域名，证书链有效且未过期。
- `miniapp/.env.production` 配置 `VITE_API_BASE_URL=https://...`，不得使用 `localhost` 或 `127.0.0.1`。
- 微信公众平台已配置 request 合法域名为后端 API 域名。
- 如后续使用文件上传或下载，需同步配置 uploadFile / downloadFile 合法域名。
- `miniapp/src/manifest.json` 和 `miniapp/project.config.json` 均保持 `urlCheck: true`。

## 3. 隐私与合规

- 已在微信公众平台提交并生效《用户隐私保护指引》。
- 小程序内已注册 `pages/privacy/privacy` 和 `pages/terms/terms`。
- 登录页必须先勾选《隐私保护指引》和《用户服务协议》才允许登录。
- 拍照识餐只在用户主动选择图片后上传图片。
- AI 顾问说明为辅助建议，不替代医生、营养师或学校管理要求。
- 用户数据删除、评价删除和数据导出流程由学校或食堂运营方确认责任人。

## 4. 产品主路径验收

- 未登录打开小程序时进入登录页。
- 微信一键登录成功后进入首页。
- 首页可以跳转到菜单、健康推荐、健康中心和智能顾问。
- 菜单页可以检索菜品、筛选价格/口味/清真、选择菜品并提交评价。
- 健康页可以进入健康推荐、拍照识餐和智能顾问。
- 我的页可以查看账号、健康档案、常用入口、隐私协议和用户协议。
- 接口异常、空数据和加载中状态均有统一提示，不出现白屏。

## 5. 构建与测试门禁

发布前必须通过：

```bash
node --test tests/wechat-auth.test.mjs
node --test tests/miniapp-readiness.test.mjs
npm run build:miniapp
```

建议发布前追加：

```bash
npm run build
node --check server/app.js
```

## 6. 微信开发者工具验收

- 导入目录：`miniapp/dist/build/mp-weixin`。
- 使用真实 AppID 预览。
- 开启“不校验合法域名”时仅用于本地调试，体验版和正式版必须关闭。
- 真机验证微信登录、首页加载、菜单检索、健康推荐、拍照识餐、智能顾问。
- 真机验证弱网、接口 401、接口 500、图片超过 5MB 等异常提示。

## 7. 发布与回滚

- 发布前记录版本号、构建时间、后端 API 域名和提交说明。
- 先上传体验版，完成真机验收后再提交审核。
- 正式发布后保留上一版后端服务兼容至少一个发布周期。
- 如发现登录、支付级主路径或数据安全问题，立即回退小程序版本并恢复上一版后端配置。
