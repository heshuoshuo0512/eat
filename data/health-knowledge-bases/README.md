# 智慧食堂健康知识库资料集

依据 `docs/健康知识库清单.md` 建立。这里保存外部可公开获取的知识库入口、已核验页面摘录、数据集说明、许可信息和项目内部资料边界。

## 目录

- `01_nutrition_guidelines/`：营养、膳食、减脂和均衡饮食指南（K12、K15、K16、K19、K21、K23、K24、K30）
- `02_food_composition/`：食物成分、营养数据集和 API（K03、K04、K13、K14、K15）
- `03_food_safety_allergy/`：食品安全、过敏原、交叉污染和转介边界（K05、K08、K25、K26）
- `04_sports_weight/`：体重管理、运动营养和训练恢复（K18-K24）
- `05_campus_operations/`：校园食堂运营、菜单、履约和反馈（K01、K06-K08、K27-K31）
- `06_rag_agent_governance/`：RAG、Agent 工具、评测、安全和隐私（K30-K36）
- `07_project_internal/`：必须由本项目生成的实时事实与审核数据边界（K01-K11、K28-K29、K32）
- `08_index_and_audit/`：总索引、覆盖矩阵、采集日志和许可审计

## 使用边界

1. 外部资料只能用于解释、规则草稿和候选营养数据；今日菜单、价格、售罄、订单、权限、用户过敏与明确忌口必须来自项目服务端事实源。
2. 外部食物成分表不能直接替代学校菜品的配方、份量和检测结果；导入时必须保留单位、份量、来源、版本、估算状态和置信度。
3. AI 只能起草，健康规则发布前必须人工审核；不用于诊断、处方或治疗承诺。
4. 资料分为：`verified_page`（本次已读取核验）、`public_entry`（公开入口待下载/核验）、`restricted`（需注册/授权）、`internal_required`（项目内部生成）。
5. 站点条款、数据许可和版本会变化；使用前重新核验。
