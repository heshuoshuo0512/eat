# 项目内部知识库：不能由外部搜索替代

依据 `docs/健康知识库清单.md` 的事实层和治理规则整理。本目录暂不填充虚构数据，仅定义采集模板和责任边界。

## 必须由项目真实数据生成

- `K01`：食堂、档口、窗口、位置、营业、拥挤度、更新时间。
- `K02`：真实菜名、别名、食材、菜系、价格、餐别、上架状态。
- `K03-K04`：真实营养、配方、份量、加工方式、估算/检测状态、置信度和版本。
- `K05`：每道菜已确认/未知的过敏原、交叉污染、清真/素食属性。
- `K06-K07`：今日菜单、供应量、售罄、实时价格、套餐与加料。
- `K08`：学校食品安全制度、投诉渠道、校医/急救渠道、地方适用法规。
- `K09-K11`：用户明确授权的健康档案、偏好、会话和记忆；禁止推断疾病。
- `K27-K29`：天气/拥挤度、订单履约、评价审核与推荐效果。
- `K32`：由真实菜品和实时菜单生成的 RAG 文档。
- `K36`：录入人、审核人、版本、有效期、变更原因、引用链、审计日志。

## 建议文件

- `source-record.schema.json`：来源与证据记录。
- `nutrition-fact.schema.json`：营养事实条目。
- `rag-document.schema.json`：RAG 文档条目。
- `evaluation-case.schema.json`：评测和红队案例。
- `school-specific-facts.template.json`：学校事实导入模板。

## 禁止

- 把外部食物数据库的数值直接写成校内菜品实测值；
- 用历史菜单、搜索结果或 RAG 文档回答当前价格、库存、售罄和订单；
- 把未知过敏原写成无过敏原；
- 把用户一次行为推断为疾病或长期健康状态；
- AI 未经审核发布健康阈值或治疗建议。
