# 管理员端功能变更说明

## 交付范围

本次重构只扩展 Web 管理员端及其后端配套能力，不新增独立数据库管理工具，不修改生产管理员的原始 SQL 入口。

## 页面变化

- `/admin` 默认进入内容审核；旧 `/admin?panel=data` 自动跳转 `/admin/catalog`。
- 新增 `/admin/catalog` 四象限目录工作区：四个固定餐饮场所桌面 2×2，移动端单列，每个场所独立滚动。
- 目录术语统一为“餐饮场所 → 餐饮分区（餐厅/楼层餐区）→ 档口 → 菜品”。
- 目录增加目录/统计切换、搜索、父级路径恢复、统计图表、权限控制和右侧编辑抽屉。
- `/admin/input` 改为分段录入，级联上下文支持保存、继续新增和返回目录。
- 菜品录入增加营养安全分组、图片上传、AI 识别预填、CSV 预览确认和导入失败重试。
- 评价管理更名为内容审核，增加帖子审核页签和重新审核流程。
- `/agent` 精简为运营智能体调试界面，隐藏原始 Schema、载荷、凭证和完整 SSE 日志。
- `/admin/ai` 拆分提供商配置、运行监控、租户与额度，并支持密钥保留/显式清空和日志分页。

## 行为和安全变化

- 所有目录写入执行权限检查、租户隔离、父级校验和审计记录。
- 新建档口必须直属餐饮分区；历史子档口保留显示并支持迁移。
- 父子档口禁止跨租户、跨食堂、自关联、循环和三级嵌套。
- 菜品变更自动使排行和检索索引失效；批量导入失败整批回滚。
- 带评分帖子通过审核后由后端创建/恢复正式评价，重复通过不会重复计入排行。
- 移动端审核表格、AI 日志和导航遮挡已适配；动效尊重 `prefers-reduced-motion`。

- `npm.cmd run build`、`git diff --check`、`node --check server/app.js` 通过。
- 全量基线为 `607` 项，`591` 通过、`16` 个既有 Agent/检索/今日菜单失败；本次管理员专项未增加失败。
