# 后端接口 Apifox 新增与变更清单

更新时间：2026-07-22
目标分支：`main`

## 使用说明

- API 前缀为 `/api`，生产地址为 `http://101.34.216.33/api`。
- 除健康检查、登录、注册、基础公开目录外，接口使用 `Authorization: Bearer <token>`。
- 学生接口按租户隔离；审核中的内容只对作者或管理员可见。
- OpenAPI 文件已同步到 `openapi/smart-canteen.yaml`，可以直接导入 Apifox。

## 学生端新增/变更

| 方法 | 路径 | 权限 | 用途 | 关键参数 |
| --- | --- | --- | --- | --- |
| `GET` | `/api/reviews` | 登录 | 菜品/食堂评价总览 | `targetType`、`canteenId`、`stallId`、`dishId`、`sort=rating_desc\|rating_asc\|latest`、`limit`、`offset` |
| `POST` | `/api/reviews` | `review:create` | 提交评价 | `targetType`、`targetId`、`rating(1-5)`、`content` |
| `GET` | `/api/posts` | 登录 | 查看已通过帖子和自己的审核状态 | `targetType`、`canteenId`、`dishId`、`limit`、`offset` |
| `POST` | `/api/posts` | `post:create` | 发布校园帖子 | `targetType`、`targetId`、`content`、可选 `imageUrl`、菜品可选 `rating` |
| `PUT` | `/api/health/profile` | 登录 | 保存健康档案并刷新推荐 | `goal`、`budgetMax`、`mealType`、`taste`、`avoid`、`allergies`、营养关注等 |
| `GET` | `/api/dishes` | 公开 | 菜品兼容列表 | `keyword`、`maxPrice`、`taste`、`halalOnly` |
| `POST` | `/api/dishes/search` | 公开 | 语义/规则菜品检索 | `query`、`filters`、`sort`、`limit`、`offset` |
| `GET` | `/api/dishes/:id` | 公开 | 菜品详情、档口、食堂、评价 | 路径参数 `id` |

评价总览返回 `reviews`、`total`、`limit`、`offset` 和 `summary`。只返回 `status=approved` 的公开评价。

帖子审核通过后，如果是菜品帖子且填写了评分，服务端使用固定 ID `post-review-<postId>` 创建或恢复正式评价，重复通过不会重复增加评价记录。

## 管理端新增/变更

| 方法 | 路径 | 权限 | 用途 |
| --- | --- | --- | --- |
| `GET` | `/api/admin/posts` | `post:moderate` | 按状态和食堂/档口/菜品筛选帖子 |
| `PATCH` | `/api/admin/posts/:id/status` | `post:moderate` | `approved`、`rejected`、`pending` |
| `GET` | `/api/admin/reviews` | `review:moderate` | 评价审核列表 |
| `PUT/PATCH` | `/api/admin/reviews/:id/status` | `review:moderate` | 更新评价审核状态 |
| `DELETE` | `/api/admin/reviews/:id` | `review:moderate` | 删除评价并刷新菜品评价数 |
| `GET` | `/api/admin/reviews/analytics` | `review:moderate` | 评价数量、均分、状态和分数分布 |
| `POST/PUT` | `/api/admin/dishes`、`/api/admin/dishes/:id` | `dish:write` | 菜品可写入 `regionalTaste`、`allergens` |

审核状态变化会写入审计日志；帖子评分和正式评价的状态保持同步。

## Apifox 建议数据集

导入接口后至少准备以下环境变量：

```text
baseUrl = http://101.34.216.33/api
studentToken = 登录接口返回的 token
adminToken = 管理员登录接口返回的 token
dishId = bootstrap 或 dishes 接口返回的真实菜品 ID
canteenId = bootstrap 或 canteens 接口返回的真实食堂 ID
stallId = bootstrap 或 stalls 接口返回的真实档口 ID
```

测试顺序建议为：登录 -> bootstrap -> 菜品/食堂目录 -> 发布评价或帖子 -> 管理员审核 -> 再次查询公开列表。
