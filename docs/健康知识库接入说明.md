# 健康知识库接入说明

健康知识库位于 `data/health-knowledge-bases/`。服务端只导入可检索的 Markdown 摘录，不把 `README.md`、`sources.md` 目录说明重复作为正文。

## 本地导入

```powershell
$env:HEALTH_KB_AUTOLOAD='1'
npm run dev:api
```

导入器会按 `HEALTH_KB_CHUNK_SIZE`（默认 900）和 `HEALTH_KB_CHUNK_OVERLAP`（默认 120）切块，并写入 `rag_documents`：

- `source_type`: `health_knowledge`
- `source_id`: 文档稳定 ID
- `metadata_json.sourceFile`: 相对知识库路径
- `metadata_json.sourceUrl`: 文档中 `来源：` URL
- `metadata_json.sourceStatus`: `verified_page`、`public_entry` 等
- `metadata_json.chunkIndex`、`citation`: 切块和引用信息

重复启动会按稳定 chunk ID upsert，不会重复插入。没有 AI embedding key 时使用确定性的本地 embedding；配置 AI provider 后会优先尝试远程 embedding，失败仍回退本地实现。

## 生产配置

`docker-compose.yml` 默认开启 `HEALTH_KB_AUTOLOAD=1`。知识库目录必须被镜像复制到 `/app/data/health-knowledge-bases`。如需关闭启动导入，设置 `HEALTH_KB_AUTOLOAD=0`，再通过 `loadHealthKnowledgeBase(db)` 执行受控导入。

## 边界

健康文档只用于解释和安全提示。今日菜单、价格、库存、订单、权限、用户过敏与忌口仍必须来自服务端事实源；外部资料不能替代学校制度、菜品配方或医疗建议。
