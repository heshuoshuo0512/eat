# 管理员端接口文档

版本：2026-07-23
认证：除公开读取接口外，均需 `Authorization: Bearer <token>`。

## 目录管理

### `GET /api/admin/catalog/tree`

权限：`audit:read`、`canteen:write`、`stall:write` 或 `dish:write` 任一。

查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `regionId` | string | 固定区域 ID：`campus-main`、`north-zone`、`south-zone`、`east-zone`。 |
| `canteenId` | string | 餐饮场所/餐饮分区 ID。 |
| `stallId` | string | 档口 ID。 |
| `q` | string | 搜索名称、品类或菜品；命中时返回完整父级路径。 |
| `include` | string | `summary` 或 `dishes`，默认返回层级和统计。 |
| `limit` / `offset` | number | 分页参数。 |

返回结构：

```json
{
  "regions": [{
    "id": "campus-main",
    "name": "中央餐饮区",
    "position": "top-left",
    "missing": false,
    "venueType": "mixed_food_hall",
    "areaType": "restaurant",
    "counts": { "canteens": 2, "stalls": 8, "dishes": 96 },
    "canteens": [{
      "canteen": { "id": "hall-a", "parentId": "campus-main" },
      "stalls": []
    }]
  }],
  "total": 4
}
```

接口按当前租户过滤，统一暴露 `parentId`，不暴露数据库字段 `parent_id`。

## 餐饮场所、档口和菜品

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| `POST` | `/api/admin/canteens` | `canteen:write` | 新增餐饮场所或餐饮分区。 |
| `PUT` | `/api/admin/canteens/:id` | `canteen:write` | 编辑名称、位置、营业信息和父级。 |
| `DELETE` | `/api/admin/canteens/:id` | `canteen:delete` | 删除；固定场所和存在下级数据时返回冲突。 |
| `POST` | `/api/admin/stalls` | `stall:write` | 新增直属餐饮分区的档口。 |
| `PUT` | `/api/admin/stalls/:id` | `stall:write` | 编辑档口，历史子档口可迁移为直属档口。 |
| `DELETE` | `/api/admin/stalls/:id` | `stall:delete` | 叶子档口可删除；存在子档口返回 `STALL_HAS_CHILDREN`。 |
| `POST` | `/api/admin/dishes` | `dish:write` | 新增菜品，档口必须属于同租户有效餐饮分区。 |
| `PUT` | `/api/admin/dishes/:id` | `dish:write` | 编辑菜品并同步检索索引、使排行失效。 |
| `DELETE` | `/api/admin/dishes/:id` | `dish:delete` | 将菜品隐藏并移除检索文档。 |

常见层级错误码：`CANTEEN_PARENT_SELF`、`CANTEEN_PARENT_NOT_FOUND`、`STALL_PARENT_SELF`、`STALL_PARENT_NOT_FOUND`、`STALL_PARENT_CROSS_CANTEEN`、`STALL_PARENT_NESTED`、`STALL_HAS_CHILDREN`、`DISH_STALL_AREA_REQUIRED`。

## 批量导入和上传

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| `POST` | `/api/admin/dishes/import/preview` | `dish:bulk_import` | 解析 CSV，校验区域→餐饮分区→档口路径、必填字段、价格和重复 ID。 |
| `POST` | `/api/admin/dishes/import/confirm` | `dish:bulk_import` | 确认预览结果，整批事务写入，失败全部回滚。 |
| `POST` | `/api/admin/dishes/import` | `dish:bulk_import` | 兼容旧 JSON 批量导入入口。 |
| `POST` | `/api/admin/dishes/vision-import` | `dish:write` | AI 图片识别，仅返回待确认表单建议。 |
| `POST` | `/api/uploads` | 登录用户 | 上传菜品或帖子图片并返回公开 URL/存储元数据。 |

## 内容审核

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| `GET` | `/api/admin/reviews` | `review:moderate` | 按 `status`、`targetType`、场所、分区、档口、菜品筛选并分页。 |
| `PATCH`/`PUT` | `/api/admin/reviews/:id/status` | `review:moderate` | `approved`、`rejected`、`pending` 状态变更。 |
| `GET` | `/api/admin/posts` | `post:moderate` | 帖子待审核和历史列表。 |
| `PATCH` | `/api/admin/posts/:id/status` | `post:moderate` | 帖子审核；评分帖子同步正式评价，禁止重复创建。 |

帖子审核、关联评价、排行失效和审计记录在同一事务中完成。返回中包含作者、正文、单图、关联对象和评价同步状态。

## 检索、智能体和 AI 配置

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| `GET` | `/api/admin/retrieval/status` | `audit:read` | 返回文档数、失败数、索引版本和最近运行状态。 |
| `POST` | `/api/admin/retrieval/reindex` | `dish:write` | 重建租户索引，可清理孤儿文档，写入审计日志。 |
| `POST` | `/api/agent/assistant` | `agent:use` | 运营智能体任务、追问、计划、引用和待确认动作。 |
| `GET` | `/api/deployment/readiness` | 管理员 | 返回运行时、数据库迁移和 AI 配置就绪状态，不返回密钥。 |
| `GET`/`PUT` | `/api/admin/ai-settings` | `ai:configure` | 读取/保存提供商配置；空 key 保留旧密钥。 |
| `DELETE` | `/api/admin/ai-settings` | `ai:configure` | 显式清空 AI 配置。 |

统一错误结构：

```json
{
  "error": {
    "code": "STALL_PARENT_CROSS_CANTEEN",
    "message": "子档口与父档口不属于同一食堂"
  }
}
```
