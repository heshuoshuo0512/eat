# 管理员端目录与接口交付总结

更新时间：2026-07-23
适用范围：Web 管理员端、管理员配套 API 与数据库；不包含微信小程序业务改造。

## 1. 本次交付内容

### 管理员端页面

- `/admin` 无面板参数时进入“内容审核”，并保留旧的 `panel=data` 兼容跳转。
- `/admin/catalog`：餐饮目录管理。桌面端固定四个餐饮场所 2×2 展示，移动端单列；每个场所内部独立滚动，避免目录数量增长撑高页面。
- 目录统一使用“餐饮场所 → 餐饮分区（餐厅/楼层餐区）→ 档口 → 菜品”术语。综合餐饮楼显示餐厅，其他场所显示楼层餐区。
- 目录支持搜索、展开/收起、数量统计、目录/统计切换、横向条形统计图、URL 上下文恢复和右侧编辑抽屉。
- `/admin/input`：按“餐饮场所、餐饮分区、档口、菜品、批量导入”分段录入；级联上下文写入 URL，支持保存、保存并继续、保存后返回目录。
- 菜品录入整合图片上传和 AI 图片识别预填；识别结果必须人工确认。CSV 采用预览→校验→确认，两阶段导入；JSON 保留在高级导入区。
- 内容审核页提供评价审核/帖子审核页签、筛选、分页、通过、驳回和重新审核；带评分帖子由后端同步正式评价，前端只展示同步结果。
- `/agent` 定位为运营智能体调试页，显示回答、意图、角色、执行计划、风险、可信度、工具成功率、引用、耗时和待确认操作；工具结果仅显示脱敏摘要。
- `/admin/ai` 拆分提供商配置、运行监控、租户与额度；空 API Key 保存会保留旧密钥，只有显式清空才删除；日志默认分页且只显示必要字段。

### 权限

- 目录读取：`audit:read` 或 `canteen:write`、`stall:write`、`dish:write`。
- 餐饮场所：`canteen:write` / `canteen:delete`。
- 档口：`stall:write` / `stall:delete`。
- 菜品：`dish:write` / `dish:delete`。
- 批量导入：`dish:bulk_import`。
- 内容审核：`review:moderate`、`post:moderate`。
- 运营智能体：`agent:use`；AI 配置：`ai:configure`；租户与额度：`tenant:manage`。

## 2. 数据库改动

### SQLite

`server/database.js` 的新库初始化和旧库回填均保持幂等，统一将数据库列映射为 camelCase 返回。

- `canteens.parent_id`：表达固定餐饮场所与餐饮分区的父子关系；`canteen_type` 自动归一为 `primary/sub`。
- `stalls.parent_id`：可空自关联字段，保留历史子档口兼容；建立 `(tenant_id, parent_id)` 索引。
- `campus_posts`：保存租户、作者、关联食堂/菜品、正文、单图、可选评分、审核状态、关联评价和时间字段。
- `audit_logs.metadata_json`：保存结构化审核元数据，包括前后状态、目标类型/目标 ID、关联评价 ID 等。
- 现有 `dishes`、`reviews`、`uploads`、`data_import_batches`、检索索引和排行数据继续复用，不新增重复业务表。

### PostgreSQL 迁移

由 `server/migrations.js` 按版本执行，失败回滚，重复启动幂等：

- `005_campus_posts.sql`：校园帖子表和索引。
- `006_admin_stall_hierarchy.sql`：`stalls.parent_id` 外键及租户父级索引。
- `007_admin_audit_metadata.sql`：`audit_logs.metadata_json`。
- `008_retrieval_pgvector.sql` 及后续运行字段：检索索引兼容管理员重建和状态监控。

父子档口校验规则：父档口必须与子档口同租户、同餐饮分区，父档口只能是一级档口；禁止自关联、循环和三级嵌套。存在子档口的一级档口删除返回冲突；历史子档口允许迁移为餐饮分区直属档口，旧数据不丢失。

## 3. 管理员 API

### 目录与录入

| 方法 | 接口 | 说明 |
| --- | --- | --- |
| `GET` | `/api/admin/catalog/tree` | 返回固定四区、餐饮分区、档口、统计；支持 `regionId`、`canteenId`、`stallId`、`q`、`include`、`limit`、`offset`；统一返回 `parentId`、`venueType`、`areaType`。 |
| `POST/PUT/DELETE` | `/api/admin/canteens[/:id]` | 餐饮场所/餐饮分区写入和删除，执行租户隔离与父级校验。 |
| `POST/PUT/DELETE` | `/api/admin/stalls[/:id]` | 档口写入、历史层级迁移和删除；禁止非法父链。 |
| `POST/PUT/DELETE` | `/api/admin/dishes[/:id]` | 菜品写入、隐藏删除、检索文档同步和排行失效。 |
| `POST` | `/api/admin/dishes/import/preview` | CSV 解析、层级校验、重复 ID 检查，返回预览和错误行。 |
| `POST` | `/api/admin/dishes/import/confirm` | 确认 CSV 导入；事务失败时整批回滚。 |
| `POST` | `/api/admin/dishes/import` | 兼容旧批量导入入口，仍受 `dish:bulk_import` 保护。 |
| `POST` | `/api/admin/dishes/vision-import` | AI 图片识别，仅返回预填建议，不绕过人工确认。 |
| `POST` | `/api/uploads` | 复用现有图片上传接口。 |

### 审核、智能体和监控

| 方法 | 接口 | 说明 |
| --- | --- | --- |
| `GET` | `/api/admin/reviews` | 按状态、类型、餐饮场所、分区、档口、菜品筛选和分页，返回完整关联对象。 |
| `PATCH/PUT` | `/api/admin/reviews/:id/status` | 评价通过、驳回、重新审核。 |
| `GET` | `/api/admin/posts` | 管理员查看帖子待审核和历史记录。 |
| `PATCH` | `/api/admin/posts/:id/status` | 帖子审核；帖子状态、关联评价同步、排行失效和审计在同一事务中完成。 |
| `GET` | `/api/admin/retrieval/status` | 返回检索文档数、失败数和最近索引状态。 |
| `POST` | `/api/admin/retrieval/reindex` | 按租户重建检索索引，支持清理孤儿文档并记录审计。 |
| `GET/POST` | `/api/deployment/readiness`、`/api/agent/assistant` | 复用现有部署检查和运营智能体服务，不新增推荐服务。 |

所有写入接口均执行权限校验、租户过滤、业务校验、审计记录；错误返回 `error.code` 和可读 `error.message`，便于前端区分权限、校验、冲突和网络错误。
