# 后端现有未文档化接口补充

更新时间：2026-07-22

这份文档补充登录、Agent、个人记录和运行状态接口。它们不是本轮新增数据库表，但前端已经依赖，建议与新增接口一起录入 Apifox。

## 认证与初始化

### `POST /api/auth/login`

请求：`{ "username": "...", "password": "..." }`。返回 `user`、JWT `token` 和完整 `state`。

### `POST /api/auth/register`

请求至少包含 `username`、`password`，可选 `nickname`。公开注册始终创建 `student`，客户端传入的 `role` 会被忽略。返回 `201`、用户、JWT 和初始化状态；用户名重复返回 `409`。

### `GET /api/bootstrap`

可匿名访问。返回 `session`、`canteens`、`stalls`、`dishes`、公开 `reviews`、登录用户的 `profile` 和 `dishPreferences`。登录后使用 Bearer token 获取个人状态。

### `GET /api/health`

返回 `{ "ok": true }`，用于部署探活，不需要登录。

## Agent 与推荐

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET/POST` | `/api/recommend` | 兼容推荐入口；POST 支持 `profileOverride` 和 `options` |
| `POST` | `/api/agent/assistant` | 意图路由、工具执行、引用、可信度和待确认操作 |
| `GET/PUT/DELETE` | `/api/agent/memory` | 读取、保存或清除个人 Agent 记忆 |
| `GET` | `/api/agent/actions` | 查看当前用户待确认/历史 Agent 操作 |
| `POST` | `/api/agent/actions/:id/confirm` | 确认高风险 Agent 操作 |
| `POST` | `/api/agent/actions/:id/reject` | 拒绝高风险 Agent 操作 |
| `POST` | `/api/recommend/plan` | 生成 1、3 或 7 天健康餐计划 |
| `GET` | `/api/rankings` | 菜品、档口和食堂排行 |

推荐接口的价格、库存、售罄、供应时间、清真和过敏原约束以 SQL 数据为准；RAG/Agent 只能补充解释，不能虚构菜品或覆盖硬约束。

## 收藏、吃过与逐张揭晓

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/preferences/dishes` | 当前用户收藏、吃过和揭晓次数 |
| `PUT` | `/api/preferences/dishes` | 请求 `{ dishId, favorite }` 更新收藏 |
| `POST` | `/api/preferences/dishes/:id/drawn` | 记录逐张揭晓 |
| `POST` | `/api/preferences/dishes/:id/eaten` | 记录吃过 |

## 菜单、订单和上传

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/menus/today` | 按 `mealType`、`date` 获取今日菜单和供应状态 |
| `POST` | `/api/orders` | 真实创建订单；今日点餐预览页当前不调用此接口 |
| `GET` | `/api/orders` | 当前用户订单和取餐码 |
| `POST` | `/api/uploads` | 登录用户上传单张图片，帖子只能引用自己的上传 URL |
| `POST` | `/api/vision/meal-analyze` | 图片识别并基于真实菜品库匹配，需 Agent 权限和 AI 配置 |

## 管理端与运维

管理端还使用 `/api/admin/menus`、`/api/admin/menu-items/:id/supply`、`/api/admin/users`、`/api/admin/audit-logs`、`/api/admin/ai-settings`、`/api/admin/ai-usage`、`/api/admin/retrieval/status` 和 `/api/admin/retrieval/reindex`。这些路径的权限和请求体已写入 `openapi/smart-canteen.yaml`。

## 错误约定

失败响应统一包含 `error`，服务端会尽量附带 `requestId`。常见状态码：`400` 参数错误、`401` 未登录、`403` 权限不足、`404` 资源不存在、`409` 唯一字段冲突、`429` 登录限流或 AI 配额耗尽。
