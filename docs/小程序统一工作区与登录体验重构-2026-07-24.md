# 小程序统一工作区与登录体验重构总结

日期：2026-07-24

## 1. 本次完成内容

本次只改造微信小程序，不修改 Web 学生端、管理员端、微信小程序后端或数据库结构。

### 找菜与智能推荐

- 将找菜和智能推荐合并到 `pages/dishes/dishes` 同一工作区。
- 顶部分段控件直接切换 `search/recommend` 内容，不再进入带返回键的推荐页面。
- 搜索结果默认只显示排名第一道菜，点击“查看全部”后展开其余排名菜品。
- 推荐结果同样默认显示第一道菜，保留完整排名展开、快捷提问、对话、饮食记忆、引用、可信度和待确认操作。
- 保留 `/pages/recommend/recommend` 作为旧链接兼容入口，自动切换到找菜页的推荐模式。

### 社区与评价

- 帖子和菜品评价继续使用同一个社区页面和分段控件。
- 首页入口可以直接指定帖子或评价页签。
- 页签切换、关键词搜索和筛选只在必要时刷新，避免每次返回 tab 都重复请求。
- 保留帖子和评价关键词搜索、280ms 防抖、分页加载及审核状态展示。

### 首页与登录

- 首页增加“校园帖子 / 菜品评价”快捷入口。
- 首页找菜和推荐入口统一通过共享状态切换工作区模式。
- 登录页改为全屏海报背景加居中登录卡，消除底部大块空白。
- 保留微信一键登录、演示账号登录、账号密码登录、隐私协议校验、加载禁用和错误提示。

## 2. 复用的接口

本次没有新增 API，全部复用现有接口：

| 功能 | 方法与路径 | 用途 |
| --- | --- | --- |
| 登录 | `POST /api/auth/login` | 账号密码登录 |
| 微信登录 | `POST /api/auth/wechat-login` | 微信授权登录 |
| 学生初始化 | `GET /api/bootstrap` | 获取用户、食堂、档口、菜品和基础评价 |
| 菜品检索 | `POST /api/dishes/search` | 关键词、预算、口味、忌口和供应检索 |
| 推荐初始化 | `GET /api/recommend` | 根据健康档案加载推荐 |
| Agent 对话 | `POST /api/agent/assistant` | 快捷提问、多轮追问和工具执行 |
| Agent 记忆 | `GET/PUT/DELETE /api/agent/memory` | 读取、保存和清除饮食记忆 |
| Agent 操作 | `POST /api/agent/actions/:id/confirm` | 确认待执行操作 |
| Agent 操作 | `POST /api/agent/actions/:id/reject` | 拒绝待执行操作 |
| 帖子列表 | `GET /api/posts?q=` | 帖子关键词搜索和分页 |
| 评价列表 | `GET /api/reviews?q=` | 评价关键词搜索、筛选、排序和分页 |
| 菜品详情 | `GET /api/dishes/:id` | 打开菜品详情和引用来源 |

## 3. 数据库变化

本次没有数据库变化：

- 没有新增表、字段、索引或迁移文件。
- 不需要设置 `DB_MIGRATE=1` 才能支持本次小程序页面。
- 推荐、帖子、评价、记忆和登录继续使用现有数据表及接口。
- 不修改 SQLite 初始化和 PostgreSQL 基线。
