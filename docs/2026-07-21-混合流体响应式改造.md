## 二、核心方案：流体为主 + 断点为辅

### 根字号流体缩放

```css
html { font-size: clamp(14px, 1.4vw, 16px); }
```

| 屏幕宽度 | 根字号 | 效果 |
|---|---|---|
| ≤1000px（手机/小平板） | 14px | 全部尺寸缩至 87.5% |
| 1000-1143px | 14→16px 平滑过渡 | 中间档位连续缩放 |
| ≥1143px（笔记本/大屏） | 16px | 跟原来一模一样 |

### 单位换算规则

- **转 rem**：padding、margin、gap、font-size、width/height、border-radius、max-width 等布局尺寸
- **保留 px**：border-width、box-shadow、outline、transform 动画值

换算公式：`原 px 值 ÷ 16 = rem 值`（如 `24px → 1.5rem`）

## 四、响应式断点体系

| 断点 | 触发 | 作用 |
|---|---|---|
| **480px** | 小手机 | 表单全宽、卡片最小 padding、登录元素极限缩小 |
| **768px** | 平板竖屏 | 毛玻璃效果取消（性能）、网格两列、登录页 hero 紧凑 |
| **900px** | 中等屏 | 营养网格提前变两列 |
| **1020px** | 平板横屏 | 侧边栏折叠为汉堡菜单 + 遮罩层、所有布局变单列、登录页竖排 |
| **1600px+** | 大桌面 | 内容区放宽至 96rem |

### 高度断点

| 断点 | 触发 | 作用 |
|---|---|---|
| **≤700px** 高度 | 矮屏幕 | 隐藏 hero-copy 和 metrics 卡片，登录卡 98vh |
| **≤500px** 高度 | 极矮屏幕 | 隐藏 eyebrow，表单极限缩小 |

## 五、滚动架构调整

```
html, body → overflow: hidden（永不滚动）
  └─ #app → height: 100%（纯容器）
       ├─ .login-landing → 94vh + 上下各 3vh margin（登录页，无滚动条）
       └─ .shell → height: 100% + overflow-y: auto（主应用在此滚动）
```

- body 不再产生滚动条
- 登录页卡片上下有间距，内容精确适配视口
- 主应用的滚动由 `.shell` 承载

## 六、登录页竖屏适配

### 布局变化（≤1020px）

- 横屏双列 → 竖屏单列（`grid-template-columns: 1fr`）
- 上面 hero 模块 `auto` 高度（紧凑），下面 login-panel `1fr`（占满剩余空间）
- 表单登录↔注册滑动逻辑同步横屏（`left: 3%` ↔ `calc(97% - 宽度)`）

### 内容缩放

- 品牌图标、标题字号、指标卡片、表单元素在窄屏下逐级缩小
- 矮屏幕自动隐藏次要文案和指标卡片

---

## 七、设计决策记录

1. **不做全量 px→rem**：border、shadow、outline、transform 保留 px，避免边框变细/阴影失调/动画异常
2. **不做移动优先**：保持桌面优先的 CSS 结构（`max-width` 降级），减少改动风险
3. **不引入 Tailwind/Container Queries**：保持现有自定义 CSS 体系，降低学习成本和侵入性
4. **滚动交给 .shell**：body 永不出滚动条，避免登录页滚动条问题反复出现
