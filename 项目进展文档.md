# 当前进展

## 项目栈
Vue 3 + Node.js + SQLite，企业级智慧食堂平台。已具备多租户、菜品/档口/食堂管理、菜单（draft/published/archived）、订单/支付/取餐码/库存/销量回滚、营业分析、RAG 推荐、Agent 中枢。

## 本轮 Agent 治理升级 - 已完成内容

### 后端（`server/app.js`）

**✅ 工具权限与审计加固**
- `assertAgentToolAllowed(tool, user)` — 每个 Agent 工具执行前做 permission/role 校验
- `runAgentTool(registry, name, user, steps, toolResults, fn)` — 统一工具执行包装，内嵌权限检查
- `runAgentTool` 所有 6 个调用点已传入 `user` 参数
- `sanitizeAgentPayload(value)` — 递归脱敏 `/api.?key|secret|token|password|credential` 字段
- `hashAgentPayloadJson(payloadJson)` — SHA-256 载荷哈希
- `defaultAgentActionExpiry()` — 默认 30 分钟过期
- `createAgentAction` — 存储脱敏载荷、计算 payloadHash、设置 expiresAt
- `confirmAgentAction` — 校验过期时间 + payload hash 一致性
- `rejectAgentAction` — 返回 payloadHash / expiresAt
- `agentSessionEvents` — 查询包含 payload_hash、expires_at

**✅ 记忆治理 API（3 endpoints）**
- `GET /api/agent/memory` — 当前用户记忆（按 tenant + user 隔离）
- `PUT /api/agent/memory` — 更新记忆（`normalizeAgentMemoryPayload` 过滤键名/类型）
- `DELETE /api/agent/memory` — 清空当前用户记忆

**✅ Eval 系统 API（5 endpoints）**
- `GET /api/agent/eval-cases` — 列出演算子（运营角色）
- `POST /api/agent/eval-cases` — 创建用例（运营角色）
- `PUT /api/agent/eval-cases/:id` — 更新用例（运营角色）
- `DELETE /api/agent/eval-cases/:id` — 删除用例（运营角色）
- `POST /api/agent/eval-cases/:id/run` — 运行用例、评分、记录结果
- `rowToAgentEvalCase` / `scoreAgentEvalCase` — 行映射和评分函数
- 学生角色所有 Eval 端点返回 403

**✅ 部署就绪 API**
- `GET /api/deployment/readiness` — 检查 agent/runtime/schema；`aiKeysConfigured: false` 保证不泄漏 AI key

**✅ 原生工具调用回退**
- `selectAgentToolCalls({ query, registry, user })` — 尝试调用 AI Provider 原生 Function Calling；失败时回退 deterministic 模式
- `agentToolFunctions(user)` / `toolRouting` — 注入到 runCanteenAgent 结果
